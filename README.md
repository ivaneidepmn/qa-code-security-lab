# 🔐 QA Code Security Lab

Hands-on project focused on automated code security and quality analysis using **GitHub Actions** and **CodeQL**.

This repository demonstrates practical implementation of **Static Code Analysis**, secure CI pipeline configuration and proactive quality practices aligned with modern **Quality Engineering** and **DevSecOps culture**.

---

## 🎯 Project Objective

The main goal of this project is to simulate a real-world scenario where a QA Engineer contributes to **preventive quality assurance**, integrating automated security analysis into the software delivery pipeline.

This approach supports:

- Early detection of vulnerabilities
- Risk reduction before production deployment
- Strengthening software reliability and maintainability
- Promoting Shift Left Testing strategy

---

## ⚙️ Technologies and Practices Applied

- GitHub Actions (CI Pipeline)
- CodeQL Security Analysis
- Static Code Analysis
- Secure Software Development Practices
- Quality Engineering mindset
- DevSecOps culture

---

## 🔄 CI Pipeline Flow

1. Developer pushes code to repository  
2. GitHub Actions workflow is triggered  
3. CodeQL performs automated security scanning  
4. Analysis results are generated and reported  
5. QA Engineer evaluates risks and supports quality decision-making
